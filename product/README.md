# Product Documentation — Delivery Clarity

This folder contains the living product documentation for Delivery Clarity.

| Document | Description | Version |
|---|---|---|
| [BRD.md](./BRD.md) | Business Requirements Document — business objectives, stakeholders, requirements, personas, risks | 1.0 |
| [SRS.md](./SRS.md) | Software Requirements Specification — functional and non-functional requirements, API spec, data model, acceptance criteria | 1.0 |

## How to keep these updated

Both documents are **living documents**. Update them when:
- A new feature is requested or built
- A requirement changes
- A known limitation is resolved
- A new stakeholder is identified
- Architecture decisions are made

Bump the version number in the Document Control section and add a row to the revision history table when making significant changes.
